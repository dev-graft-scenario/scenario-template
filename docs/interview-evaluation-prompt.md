```
## 평가 기준

### 평가 시 고려사항:
1. **README 요구사항 준수도** - 명시된 기능과 제약사항 구현 여부
2. **아키텍처 적절성** - 문제 복잡도에 맞는 설계 선택
3. **코드 품질** - 가독성, 유지보수성, 테스트 커버리지
4. **기술 숙련도** - 사용 기술에 대한 이해도와 활용 능력
5. **시간 효율성** - 제한된 시간 내 우선순위 설정과 완성도
6. **확장성 고려** - 향후 요구사항 변화에 대한 대비
7. **문제 해결 능력** - 제약 조건 하에서의 창의적 해결책

### 점수 기준:
- 90-100: A등급 (적극 채용 권장)
- 80-89: B등급 (채용 권장)
- 70-79: C등급 (조건부 채용 검토)
- 60-69: D등급 (추가 면접 필요)
- 60 미만: F등급 (불합격)

### 질문 난이도 조정:
- **주니어 (1-3년차):** 기본 구현과 코드 품질 중심
- **미들 (3-5년차):** 설계 패턴과 성능 최적화
- **시니어 (5년차+):** 아키텍처 결정과 시스템 설계, 기술 리딩

### 도메인별 특화 질문:
- **웹 서비스:** API 설계, 보안, 캐싱
- **데이터 처리:** 대용량 처리, 배치, 실시간 처리
- **인프라:** 배포, 모니터링, 장애 대응
- **AI/ML:** 모델 성능, 데이터 파이프라인
```

## 사용 예시

```
# 사용 방법
1. 프로젝트 루트에서 README.md 파일이 있는지 확인
2. 위 프롬프트를 Ai(Claude, Gemini, GPT, Cursor 등)에 입력
3. 자동으로 README 기반 평가 문서 생성
4. 면접관은 생성된 질문으로 기술 면접 진행

# 커스터마이징
- 포지션별로 질문 난이도 조정
- 회사 기술 스택에 맞춰 질문 추가
- 평가 기준을 회사 문화에 맞게 수정
```
